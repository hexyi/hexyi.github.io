    接口API

    1.电子账户开户

    接口方法名：accountOpen
    功能处理描述：根据用户提供的开户资料，申请开通银行电子账户
    调用方式：dubbo
    接口类名：com.vccount.trade.facade.account.AccountFacade
    http方式：/app/account/accountOpen.security

    请求参数说明：com.vccount.trade.facade.dto.account.AccountReqDTO

    名称 类型 必填 说明
    channel String Y 渠道 默认蔚捷(WJ) 冠群(GQ)
    customerName String Y 姓名
    sex String Y 性别(M男 F女)
    indentityType String Y 证件类型01身份证
    indentityNo String Y 证件号
    idCardPicFName String Y 身份证照片正面路径
    idCardPicBName String Y 身份证照片反面路径
    bankName String N 开户行名称
    bankCard String Y 银行卡号
    phone String Y 银行预留手机号

    返回参数说明：com.vccount.trade.facade.dto.account.AccountResDTO

    名称 类型 必填 说明
    code String Y 结果码(请参照下面错误码表)
    msg String Y 结果描述
    userId String Y 用户Id
    subAcctNo String Y E账号

    服务级错误码参照(error_code)：

    错误码 说明
    600 操作成功(成功)
    601 参数不能为空(可再次请求处理)
    602 参数不正确(可再次请求处理)
    603 参数不正确(可再次请求处理)
    604 用户不存在(可再次请求处理)
    605 渠道非法(可再次请求处理)
    606 信息异常(需要运营介入)
    607 内部服务器错误(需要运营介入)
    608 加解密异常(需要运营介入)
    609 不能重复提交(需要运营介入)
    610 编码转换失败(需要运营介入)
    611 已经存在(重新请求)
    612 远程调用异常(需要运营介入)
    613 请求频繁(稍后再试)
    614 暂无记录(稍后再试)
